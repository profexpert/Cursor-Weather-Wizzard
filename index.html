Шпаргалка по Git

Список команд basic and advanced:

  1. git config --global user.name blabla - це команда яка додає ваше ім'я користувача яке буде в подальшому використовуватись в коммітах.
  2. git config --global user.email blabal@bla - це команда яка додає ваш e-mail який буде в подальшому використовуватись в коммітах.
  3. git init - Команда git init створює новий репозиторій Git. З її допомогою можна перетворити існуючий проект без керування версіями на репозиторій Git або ініціалізувати новий порожній репозиторій.
  4. git status - Команда git status показує стан файлів у робочому каталозі та індексі: які файли змінені, але не додані в індекс; які очікують коміти в індексі. На додаток до цього виводяться підказки про те, як змінити стан файлів.
  5. git add - Команда git add - це перша команда в ланцюжку операцій, що наказує Git "зберегти" знімок поточного стану проекту в історії коммітів. Коли git add використовується як окрема команда, вона переносить очікувані зміни з робочого каталогу в розділ проіндексованих файлів.
  6. git commit - створює новий коміт, в основі якого лежать існуючі файли, додаючи зміни тільки для зазначених файлів.
  7. git log - це команда щоб переглягути історію комміта.
  8. git checkout - дозволяє переміщатися між гілками, створеними командою git branch
  9. git branch - дозволяє створювати, переглядати, перейменовувати та видаляти гілки.
  10. git push - використовується для вивантаження вмісту локального репозиторію у віддалений репозиторій. Вона дозволяє передати коміти з локального репозиторію у віддалений.
  11. git pool - використовується для видалення та завантаження вмісту віддаленого репозиторію та негайного оновлення локального репозиторію цим вмістом.
  12. git merge - виконує злиття окремих напрямів розробки, створених за допомогою команди git branch в єдину гілку.
  13. git clone - Використовується для отримання копії існуючого Git-репозиторію, наприклад, проекту, в який ви хочете зробити свій внесок
  14. git revert - безпечний спосіб скасування операції без втрати історії коммітів. Команда скасовує дії минулих коммітів, створюючи новий, що містить всі скасовані зміни. Ця команда корисна, коли ви вже запушили зміни у віддалений репозиторій, оскільки вона зберігає незайманим вихідний коміт.
  15. git reset - це потужна команда, яка використовується для скасування локальних змін у репозиторії Git. Git reset працює з «трьома деревами Git»: історією коммітів (HEAD), розділом проіндексованих файлів та робочим каталогом.
  16. git rebase - ще один спосіб перенести зміни з однієї гілки до іншої. Rebase стискає всі зміни в один патч. Потім він інтегрує патч у цільову гілку. На відміну від злиття, переміщення перезаписує історію, тому що вона передає завершену роботу з однієї гілки до іншої.
  17. git cherry-pick - бере зміни, що вносяться одним коммітом, та намагається повторно застосувати їх у вигляді нового комміту в поточній гілці. Ця можливість корисна в ситуації, коли потрібно забрати парочку коммітів з іншої гілки, а не зливати гілку цілком з усіма змінами, що внесені до неї.
  18. git stash - призначена для того, щоб помістити поточні зміни, які ви здійснили у файлах, в окреме сховище, та повернути файли до вихідного стану. Тобто git stash ховає зміни у файлах та зберігає ці зміни окремо, щоб потім можна було їх повернути.
  19. gitignore - потрібний для приховання файлів та папок від системи контролю версій Git. Зазвичай приховують конфігураційні файли (особливо з паролями), тимчасові файли та папки.
